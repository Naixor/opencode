## Codebase Patterns
- Use `Instance.state()` with synchronous initializer for per-instance state
- Sandbox profile generation is in src/sandbox/profile.ts, seatbelt wrapper in src/sandbox/seatbelt.ts
- Security config loaded from .opencode-security.json via src/security/config.ts
- SandboxConfig interface defined in src/sandbox/index.ts
- Tests run via: cd packages/opencode && bun run test:parallel --pattern "pattern"
- seatbelt uses POSIX ERE regex, not PCRE
- seatbelt rule evaluation is last-match-wins
- Profile generation order: builtins(deny all writes) -> allowlist rules -> deny rules
- glob-to-regex: `**` becomes `(/.+)?/` (zero or more dirs) when followed by next segment, or `/.*` when trailing
- glob-to-regex: `*` becomes `[^/]*` (no dir crossing), `?` becomes `[^/]`
- Fixed prefix realpath: split by /, stop at first segment with wildcards, realpath the prefix
