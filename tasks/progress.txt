## Codebase Patterns

- Use `SwarmState.create/read/write` in `packages/opencode/src/session/swarm-state.ts` for canonical swarm snapshot setup instead of hand-rolling `state.json` payloads.
- Route authoritative swarm/task/discussion changes through `SwarmState.mutate`; `BoardTask` and `Discussion` raw JSON files are compatibility mirrors, not the source of truth.
- Worker timeout tests should call `Swarm.status()` after seeding a `waiting` worker, because the timeout promotion to `blocked` happens when swarm state is refreshed.
