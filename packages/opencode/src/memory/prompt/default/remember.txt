The user wants to save a memory about their preferences or project conventions.

User's input: $ARGUMENTS

Your task:
1. Look at the recent conversation context to understand what "$ARGUMENTS" refers to
2. Create a clear, self-contained memory that can be understood without this conversation
3. Use the Memory system to save it

Call the memory_remember tool with:
- content: A clear, specific, concise description of the preference/convention (no pronouns like "this" or "that"); prefer one durable point and structured wording like "Topic: detail"
- category: One of "style", "pattern", "tool", "domain", "workflow", "correction", "context"
- tags: Relevant keywords for future recall

If the user's input is vague (e.g., "remember this"), use the conversation context to figure out what they mean and create a specific memory.

After saving, briefly confirm what was remembered.
