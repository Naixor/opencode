## Codebase Patterns
- Memory web UI is mounted as Hono sub-route: `.route("/memory", MemoryRoutes())` with `/app` serving HTML via `import APP_HTML from "./app.html" with { type: "text" }`
- ManagerState.register() takes array of `{ id, name, description, url, icon }` — manager dashboard fetches from `GET /manager/services`
- LogServer in `src/log/server.ts` is a standalone Bun.serve() server — needs conversion to Hono sub-route pattern
- Log-viewer assets are embedded at build time via `OPENCODE_LOG_VIEWER_ASSETS` define in build.ts
- Existing log query API (LlmLog namespace) is in `src/log/query.ts`
- LogViewerCommand CLI is in `src/cli/cmd/log-viewer.ts`, registered in `src/index.ts`
- Manager CLI command is in `src/cli/cmd/manager.ts` — already has log-viewer as a service option
