## Codebase Patterns
- Keyword detection hooks are in `src/session/hooks/detection-checking.ts`, registered via `HookChain.register(name, chainType, priority, handler)`
- Tests for hooks use `withInstance()` helper that sets up temp dir, resets HookChain, and registers hooks
- `getLastUserMessage()` extracts text from last user message (supports string and multipart content)
- To modify message content in a hook, you need a reference to the message object (not just the text) — use a separate ref-returning function
- Tests run from `packages/opencode` dir; git operations use repo root paths
- `bun run typecheck` runs from repo root; `bun run test:parallel` runs from `packages/opencode`
- TUI local context (`src/cli/cmd/tui/context/local.tsx`) uses `iife()` + closure `let` vars for transient state that shouldn't persist to `model.json`
- The variant API's `setOneShot` must use `setModelStore()` directly, not `this.set()`, to avoid clearing the one-shot flag
