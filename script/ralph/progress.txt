## Codebase Patterns
- Sandbox SBPL profile must use `(allow default)` + `(deny file-write* (subpath "/"))` approach, NOT `(deny default)`. The `(deny default)` blocks system operations (mach-lookup, dyld loading) causing SIGABRT.
- On macOS `os.tmpdir()` returns `/var/folders/...` not `/tmp`. Both need whitelisting.
- `fs.realpath()` resolves symlinks: `/tmp` → `/private/tmp`, `/var` → `/private/var`.
- Config is accessed via `await Config.get()`, NOT `Config.state.use()`.
- CLI commands use `cmd()` wrapper from `./cmd`. Register in `index.ts` with `.command(XCommand)`.
- File tools (read/write/glob/grep) use `SecurityAccess.checkAccess()` at app layer — no sandbox wrapping needed.
- MCP trust policy from `SecurityConfig.getMcpPolicy(key)`: `enforced` = sandbox, `trusted` = skip.
- TUI sync data flows: server route → SDK client → sync store → component. For non-SDK routes, use raw `fetch()`.

- **Learnings for future iterations:**
  - macOS Seatbelt `(deny default)` denies ALL operations including mach-lookup, causing SIGABRT. Must use `(allow default)` + `(deny file-write* (subpath "/"))` pattern instead.
  - `os.tmpdir()` returns `/var/folders/.../T` on macOS, different from `/tmp` → `/private/tmp`. Both paths must be whitelisted.
  - SBPL regex lines need careful paren counting: `(allow file-write* (regex "..."))` needs two closing parens.
  - sandbox-exec exit code 134 = SIGABRT (signal 6 + 128), usually means invalid SBPL profile.
  - sandbox-exec exit code 65 = Operation not permitted (sandbox policy denied the operation).
