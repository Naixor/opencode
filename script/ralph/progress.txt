## Codebase Patterns
- TUI sync context is in `packages/opencode/src/cli/cmd/tui/context/sync.tsx`, web app sync is in `packages/app/src/context/sync.tsx` — they have different implementations
- `Binary.search()` is the standard pattern for sorted array lookups across the TUI codebase (returns `{ found, index }`)
- SolidJS `produce()` from `solid-js/store` is used for immutable-style draft mutations in store updates
- `fullSyncedSessions` Set in TUI sync prevents redundant full syncs — only tracks completed full syncs, not SSE-partial population
- Web app sync already calls `sync.session.sync()` unconditionally via `createEffect` — TUI was the only place with the guard bug
- Message IDs are ULIDs — `lt()` on ID column gives chronological cursor-based pagination for free
- `MessageV2.stream()` takes `{ sessionID, before? }` object — all callers must use object form
- TUI scrollbox (`ScrollBoxRenderable`) has no onScroll callback — scroll is commanded via keybinds; check `scroll.y` after scroll commands to detect top
- SDK generated types are in `packages/sdk/js/src/v2/gen/{sdk,types}.gen.ts` — update both when adding API params; also update `packages/sdk/openapi.json`
- `Session.fork()` reads all messages from SQLite server-side — it's immune to client-side message limits and works correctly for remote-attached sessions
