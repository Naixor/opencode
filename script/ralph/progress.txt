## Codebase Patterns
- `executePostTool` pluginOutput type is `Record<string, any>` — supports both built-in tools (`{ title, output, metadata }`) and MCP tools (`{ content, metadata }`)
- For MCP tools, PostToolContext.result.output is constructed by joining text content items; the raw `result` object is passed to Plugin.trigger for backwards compat
- Hook registration files must avoid circular imports: hook files import from `./index` (HookChain), so `registerAll` lives in a separate `register.ts` file
- All hook namespaces have `register()` functions that call `HookChain.register()` internally
- Typecheck for the main opencode package: `cd packages/opencode && bun run typecheck` (or `bun run typecheck` from repo root)
- Drizzle migration generation: `cd packages/opencode && bun drizzle-kit generate --name <name>` — must manually add IF NOT EXISTS after
- New `*.sql.ts` files auto-discovered by drizzle config; new table exports must be added to `src/storage/schema.ts`
- Git paths must be relative to repo root (`/Users/bytedance/GithubWorkspace/@Naixor/opencode`), not packages/opencode
- Working directory may be `packages/opencode` due to turbo — always use absolute or repo-root-relative paths for git
- `executePostTool` writeback requires `pluginOutput` to have an `output` field — MCP result objects need a wrapper object
- `Identifier` prefixes must be registered in `src/id/id.ts` before using `Identifier.ascending(prefix)` — added `log: "log"` for llm_log tables
- `LlmLogCapture.getCurrentLogId(sessionID)` provides cross-chain access to the current llm_log_id (used by US-009, US-010, US-011)
