## Codebase Patterns
- `executePostTool` pluginOutput type is `Record<string, any>` — supports both built-in tools (`{ title, output, metadata }`) and MCP tools (`{ content, metadata }`)
- For MCP tools, PostToolContext.result.output is constructed by joining text content items; the raw `result` object is passed to Plugin.trigger for backwards compat
- Hook registration files must avoid circular imports: hook files import from `./index` (HookChain), so `registerAll` lives in a separate `register.ts` file
- All hook namespaces have `register()` functions that call `HookChain.register()` internally
- Typecheck for the main opencode package: `cd packages/opencode && bun run typecheck` (or `bun run typecheck` from repo root)
- Drizzle migration generation: `cd packages/opencode && bun drizzle-kit generate --name <name>` — must manually add IF NOT EXISTS after
- New `*.sql.ts` files auto-discovered by drizzle config; new table exports must be added to `src/storage/schema.ts`
- Git paths must be relative to repo root (`/Users/bytedance/GithubWorkspace/@Naixor/opencode`), not packages/opencode
- Working directory may be `packages/opencode` due to turbo — always use absolute or repo-root-relative paths for git
- `executePostTool` writeback requires `pluginOutput` to have an `output` field — MCP result objects need a wrapper object
- `Identifier` prefixes must be registered in `src/id/id.ts` before using `Identifier.ascending(prefix)` — added `log: "log"` for llm_log tables
- `LlmLogCapture.getCurrentLogId(sessionID)` provides cross-chain access to the current llm_log_id (used by US-009, US-010, US-011)
- Instance.state for llm-log stores `{ logId, timeStart }` — use `currentLogState().get(sessionID)` for both values
- `metadata: { callID }` is passed on all PreToolContext/PostToolContext in prompt.ts — hooks access via `ctx.metadata?.callID`
- Pre/post tool hook correlation key: `${sessionID}:${callID ?? toolName}` stored in `toolCallStartState` Instance.state
- Shared Instance.state between modules: extract to a separate `log-state.ts` file to avoid circular imports between `hooks/index.ts` and `capture.ts`
- Drizzle `.run()` on delete/insert/update returns `void` — count records before deleting if you need the count
- Zod `.optional()` schema `parse()` returns `undefined` when no input — use `parsed?.field` for safe access
- Background periodic tasks use `Scheduler.register({ id, interval, run, scope })` from `src/scheduler/index.ts`; init calls go in `src/project/bootstrap.ts`
- React packages in this monorepo: must add `"react": ["./node_modules/@types/react"]` to tsconfig paths to fix duplicate @types/react resolution (monorepo has @types/react@18.0.25 hoisted, conflicts with package-level @types/react@19.x)
- Use `react-router` v7 (not `react-router-dom`) — v7 primary package, `react-router-dom` is compat layer
- Use `createBrowserRouter` + `RouterProvider` API (not `BrowserRouter` + `Routes`) — better types and data loading support
- Recharts tooltip/label callbacks: don't annotate param types explicitly — use `(value) => ...` and cast with `Number(value)` inside to avoid strict type mismatches

## 2026-03-07 - US-021
- Implemented StatsPage with summary cards and 4 chart/table views
- Summary cards: total requests, total tokens (in/out), total cost, avg tokens/request
- Line chart: token usage & cost over time (hourly/daily) with dual Y-axes (tokens left, cost right)
- Bar chart: token consumption by model (input vs output stacked)

- Session table: per-session token totals, request count, cost, avg duration
- All views share a time range filter (1h/6h/24h/7d/30d/all)
- Each chart component fetches its own data with the appropriate `group_by` parameter
- Installed `recharts@3.8.0` as dependency
- Files changed: `packages/log-viewer/src/pages/StatsPage.tsx`, `packages/log-viewer/package.json`, `bun.lock`
- **Learnings for future iterations:**
  - Recharts `Tooltip` `formatter` and `labelFormatter` callbacks have loose types — use `(value) => ...` without explicit type annotations and cast with `Number(value)` / `String(label)` inside
  - Recharts `Pie` `label` callback: `percent` can be undefined — use `(percent ?? 0)`
  - Dual Y-axis pattern: use `yAxisId` on both `<YAxis>` and `<Line>` to associate them
  - Cost is stored as microdollars (integer) in the API — divide by 1_000_000 for display
  - Time grouping returns timestamps as strings — sort numerically with `Number(a.group)` for correct time ordering
  - Manual browser verification needed (no browser testing tools configured)
