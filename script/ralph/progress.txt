## Codebase Patterns
- Auth storage lives in `Global.Path.data` (xdg data dir) as JSON files with 0o600 permissions
- CLI subcommands use `cmd()` wrapper from `./cmd.ts` and are registered in `src/index.ts`
- Internal plugins are added to `INTERNAL_PLUGINS` array in `src/plugin/index.ts`
- Plugin type requires async function returning `Hooks` object: `Plugin = (input: PluginInput) => Promise<Hooks>`
- `chat.headers` hook runs before every LLM request, receives `(input, output)` where output has headers record
- Wellknown auth entries are stored with `type: "wellknown"` in auth.json, key is the URL
- Use `Filesystem.readJson/writeJson` for atomic JSON file operations
- Typecheck command: `bun run typecheck` (uses tsgo in opencode package, tsc in others)
